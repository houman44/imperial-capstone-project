# Datasheet for BBO Capstone Project Dataset

## Motivation

**Why was this dataset created?**  
This dataset was created to support a Black-Box Optimization (BBO) capstone project in which participants submit queries to unknown objective functions and receive scalar outputs. The task is to maximize or minimize each function across multiple rounds by accumulating observations and refining a surrogate model to recommend the next queries.

**What task does it support?**  
The dataset supports iterative Bayesian optimization: participants use the accumulated (input, output) pairs to train surrogate models (e.g. Gaussian processes) and acquisition functions (e.g. Expected Improvement) to propose new query points for the next round.

## Composition

**What does the dataset contain?**  
- **Functions:** Eight black-box functions (F1–F8) with different input dimensions:
  - F1, F2: 2D (maximize)
  - F3: 3D (minimize)
  - F4: 4D (minimize)
  - F5: 4D (maximize)
  - F6: 5D (minimize)
  - F7: 6D (maximize)
  - F8: 8D (maximize)

- **Per-function data:** For each function, a CSV with columns: `round`, `source`, `x1`–`x8` (only first N used per function), `y` (observed output).

- **Size and format:** Variable by round. As of round 8, roughly 15–25 observations per function. Format: CSV, numeric values in [0, 1] for inputs; outputs are real-valued (scale varies by function).

- **Gaps:** Early rounds may have fewer points. Some functions (e.g. F8) have sparser coverage due to higher dimensionality. No explicit missing-value encoding; all rows are complete observations.

## Collection Process

**How were the queries generated?**  
Queries were generated by: (1) fitting a Gaussian Process surrogate to all accumulated (X, y) per function; (2) using Expected Improvement (EI) as the acquisition function to score candidate points; (3) selecting the point with highest EI. For some functions, hybrid strategies were used (e.g. SVM pre-screening, gradient refinement on the GP surrogate).

**What strategy was used?**  
Strategy evolved over rounds: pure GP+EI for low-dimensional functions; GP+SVM screening for medium-dimensional; GP+gradient refinement for functions with suspected smooth or boundary optima. The acquisition parameter ξ=0.01 balanced exploration and exploitation.

**Over what time frame?**  
Data were collected across multiple submission rounds (weeks) as part of the capstone project timeline. Each round added one new (input, output) pair per function from the external evaluation platform.

## Preprocessing and Uses

**Transformations applied:**  
- Inputs constrained to [0, 1] per competition specification.  
- Outputs used as provided; no normalization in the surrogate for EI (though GP uses `normalize_y=True` internally).  
- Categorical or special encodings: none; all inputs are continuous.

**Intended uses:**  
- Training surrogate models for next-query recommendation.  
- Analyzing performance across functions and rounds.  
- Reproducing the BBO strategy and comparing with alternative approaches.

**Inappropriate uses:**  
- Generalizing to other black-box functions or domains without validation.  
- Using as a general-purpose optimization benchmark without acknowledging its capstone-specific design.  
- Treating the data as representative of real-world optimization scenarios (functions are synthetic and competition-defined).

## Distribution and Maintenance

**Where is the dataset available?**  
The dataset is stored in the project repository under `data/functions/` (function_1.csv through function_8.csv). Metadata (function specs, objectives) is in `data/metadata.json`.

**Terms of use:**  
Subject to the capstone project and platform terms. The data is for educational and portfolio purposes. External evaluation results (y values) were obtained by submitting to the capstone project portal.

**Who maintains it?**  
The project author maintains the dataset. Updates occur as new rounds are imported via `capstone_manager.py import-results`. Versioning is implicit through round numbers in the CSVs; no formal changelog is maintained.
